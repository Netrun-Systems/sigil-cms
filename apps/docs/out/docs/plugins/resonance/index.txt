9:T8e5,<h2>Overview</h2>
<p>The Resonance plugin adds content performance analytics at the block level. It tracks engagement (views, clicks, scrolls, time-on-block), supports A/B testing between block variants, and optionally uses Gemini AI to suggest content improvements.</p>
<p><strong>Required env</strong>: None (Gemini AI suggestions optional via <code>GEMINI_API_KEY</code>).</p>
<h2>Features</h2>
<ul>
<li><strong>Block-level analytics</strong> -- track views, clicks, scroll depth, and dwell time per block</li>
<li><strong>A/B testing</strong> -- create block variants and measure performance</li>
<li><strong>AI suggestions</strong> -- Gemini-powered content improvement recommendations</li>
<li><strong>Engagement scoring</strong> -- composite score combining multiple engagement signals</li>
<li><strong>Client snippet</strong> -- inject a tracking script on consumer sites</li>
</ul>
<h2>How Scoring Works</h2>
<p>The scoring engine combines multiple engagement signals into a single resonance score:</p>
<ul>
<li><strong>View rate</strong> -- what percentage of visitors see this block</li>
<li><strong>Click-through rate</strong> -- clicks on CTAs or links within the block</li>
<li><strong>Scroll depth</strong> -- how far users scroll through long blocks</li>
<li><strong>Dwell time</strong> -- time spent with the block visible in viewport</li>
</ul>
<p>Higher resonance scores indicate content that holds visitor attention.</p>
<h2>A/B Testing</h2>
<p>Create multiple variants of a block and let Resonance distribute traffic:</p>
<ol>
<li>Create a block variant through the admin panel</li>
<li>Resonance splits traffic between variants</li>
<li>After statistical significance, the winning variant is identified</li>
<li>Admin can promote the winner as the default</li>
</ol>
<h2>AI Suggestions</h2>
<p>When <code>GEMINI_API_KEY</code> is configured, Resonance can analyze underperforming blocks and suggest:</p>
<ul>
<li>Headline rewrites</li>
<li>CTA text improvements</li>
<li>Layout changes</li>
<li>Content restructuring</li>
</ul>
<h2>Client Integration</h2>
<p>The plugin generates a tracking snippet (similar to the support panel snippet) that you add to your consumer site. It automatically tracks engagement events and reports them to the Sigil API.</p>
