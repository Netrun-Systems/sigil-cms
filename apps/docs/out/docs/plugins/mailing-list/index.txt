9:T53b,<h2>Overview</h2>
<p>The Mailing List plugin provides email subscription management with CAN-SPAM and GDPR compliance built in. Subscribers are managed per-site with one-click unsubscribe tokens.</p>
<p><strong>Required env</strong>: None (Azure Communication Services is optional for broadcast sending).</p>
<h2>Features</h2>
<ul>
<li><strong>Public subscribe</strong> -- visitors can subscribe via API or newsletter blocks</li>
<li><strong>One-click unsubscribe</strong> -- token-based, no auth required (GDPR compliant)</li>
<li><strong>Broadcast sending</strong> -- send to all subscribers (requires ACS)</li>
<li><strong>Admin management</strong> -- view, search, and manage subscriber lists</li>
</ul>
<h2>Routes</h2>
<h3>Public (no auth)</h3>
<ul>
<li><code>POST /api/v1/public/subscribe/:siteSlug</code> -- subscribe an email</li>
<li><code>GET /api/v1/public/unsubscribe/:token</code> -- unsubscribe with token</li>
</ul>
<h3>Admin (auth required)</h3>
<ul>
<li><code>/api/v1/sites/:siteId/subscribers</code> -- list, manage subscribers</li>
</ul>
<h2>Admin Navigation</h2>
<p>The plugin adds a "Mailing List" item under the "Engagement" admin section.</p>
<h2>Integration with Blocks</h2>
<p>Use the <code>newsletter</code> block type to add signup forms to any page. The block automatically posts to the subscribe endpoint.</p>
