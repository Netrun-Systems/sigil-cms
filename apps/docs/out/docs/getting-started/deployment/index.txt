9:Td67,<h2>Docker Deployment</h2>
<p>Sigil ships with a multi-stage Dockerfile at <code>apps/api/Dockerfile</code>. Build context must be the repo root:</p>
<pre><code class="language-bash">docker build -f apps/api/Dockerfile -t sigil-api .
docker run -p 3000:3000 \
  -e DATABASE_URL=postgresql://user:pass@host:5432/sigil \
  -e JWT_SECRET=your-secret-here \
  sigil-api
</code></pre>
<p>The image uses <code>node:20-alpine</code>, runs on port 3000, and bundles all <code>@netrun-cms/*</code> packages.</p>
<h2>Google Cloud Run</h2>
<pre><code class="language-bash"># Build and push
docker build -f apps/api/Dockerfile -t us-central1-docker.pkg.dev/PROJECT/repo/sigil-api:latest .
docker push us-central1-docker.pkg.dev/PROJECT/repo/sigil-api:latest

# Deploy
gcloud run deploy sigil-api \
  --image us-central1-docker.pkg.dev/PROJECT/repo/sigil-api:latest \
  --port 3000 \
  --set-env-vars DATABASE_URL=...,JWT_SECRET=... \
  --region us-central1 \
  --allow-unauthenticated
</code></pre>
<h2>Azure Container Apps</h2>
<pre><code class="language-bash"># Build and push to ACR
docker build -f apps/api/Dockerfile -t myregistry.azurecr.io/sigil-api:latest .
docker push myregistry.azurecr.io/sigil-api:latest

# Update Container App
az containerapp update \
  --name sigil-api \
  --resource-group my-rg \
  --image myregistry.azurecr.io/sigil-api:latest
</code></pre>
<p>Recommended Container App settings for development:</p>
<ul>
<li><strong>Min replicas</strong>: 0 (scales to zero = no cost when idle)</li>
<li><strong>Max replicas</strong>: 1</li>
<li><strong>CPU</strong>: 0.25 vCPU</li>
<li><strong>Memory</strong>: 0.5 GB</li>
<li><strong>Estimated cost</strong>: ~$5/month</li>
</ul>
<h2>Admin Panel (Static)</h2>
<p>The admin is a Vite SPA that can be deployed to any static hosting:</p>
<pre><code class="language-bash"># Build admin
pnpm --filter @netrun-cms/admin build

# Deploy to Azure Static Web Apps
npx @azure/static-web-apps-cli deploy apps/admin/dist \
  --deployment-token "YOUR_TOKEN"
</code></pre>
<p>Or deploy <code>apps/admin/dist/</code> to Vercel, Netlify, or Cloudflare Pages.</p>
<h2>Database Requirements</h2>
<p>Sigil requires PostgreSQL. The production schema lives in <code>packages/@netrun-cms/db/src/schema.ts</code> with 7 core tables:</p>
<ul>
<li><code>cms_tenants</code> -- multi-tenant isolation</li>
<li><code>cms_sites</code> -- individual websites</li>
<li><code>cms_pages</code> -- page content with hierarchy</li>
<li><code>cms_content_blocks</code> -- composable blocks</li>
<li><code>cms_media</code> -- file management</li>
<li><code>cms_themes</code> -- design tokens</li>
<li><code>cms_users</code> -- authentication</li>
</ul>
<p>Plugins create additional tables (e.g., <code>cms_doc_categories</code>, <code>cms_store_products</code>) via their <code>register()</code> migration hooks.</p>
