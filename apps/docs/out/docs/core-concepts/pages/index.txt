<h2>Page CRUD</h2>
<p>All page routes are scoped to a site: <code>/api/v1/sites/:siteId/pages</code>.</p>
<h3>List pages</h3>
<pre><code>GET /api/v1/sites/:siteId/pages
</code></pre>
<p>Query params:</p>
<ul>
<li><code>page</code> (default: 1)</li>
<li><code>limit</code> (default: 20, max: 100)</li>
<li><code>status</code> -- filter by <code>draft</code>, <code>published</code>, <code>scheduled</code>, <code>archived</code></li>
<li><code>language</code> -- filter by language code</li>
<li><code>parentId</code> -- filter by parent (use <code>null</code> for root pages)</li>
</ul>
<h3>Create a page</h3>
<pre><code>POST /api/v1/sites/:siteId/pages
</code></pre>
<pre><code class="language-json">{
  "title": "About Us",
  "slug": "about",
  "status": "draft",
  "language": "en",
  "metaTitle": "About Our Company",
  "metaDescription": "Learn about our team and mission."
}
</code></pre>
<p>Requires role: <code>admin</code>, <code>editor</code>, or <code>author</code>.</p>
<h3>Update / Delete</h3>
<pre><code>PUT    /api/v1/sites/:siteId/pages/:id
DELETE /api/v1/sites/:siteId/pages/:id
</code></pre>
<p>Delete removes the page and all its content blocks. Requires <code>admin</code> or <code>editor</code>.</p>
<h2>Content Scheduling</h2>
<p>Schedule pages to publish or unpublish at specific times:</p>
<pre><code>PATCH /api/v1/sites/:siteId/pages/:id/schedule
</code></pre>
<pre><code class="language-json">{
  "publishAt": "2026-04-01T09:00:00Z",
  "unpublishAt": "2026-04-30T23:59:59Z"
}
</code></pre>
<ul>
<li><code>publishAt</code> in the future sets status to <code>scheduled</code></li>
<li><code>publishAt</code> in the past publishes immediately</li>
<li><code>null</code> clears the schedule</li>
</ul>
<p>Requires role: <code>admin</code> or <code>editor</code>.</p>
<h2>Translations</h2>
<p>Sigil supports multi-language content through page translations. Each translation shares the same slug but has a different language code.</p>
<pre><code>GET  /api/v1/sites/:siteId/pages/:id/translations
POST /api/v1/sites/:siteId/pages/:id/translate
</code></pre>
<p>The translate endpoint clones the page and all its content blocks with the new language code:</p>
<pre><code class="language-json">{ "language": "es" }
</code></pre>
<h2>Revisions</h2>
<p>Every page maintains a revision history:</p>
<pre><code>GET  /api/v1/sites/:siteId/pages/:pageId/revisions
GET  /api/v1/sites/:siteId/pages/:pageId/revisions/:revisionId
POST /api/v1/sites/:siteId/pages/:pageId/revisions/:revisionId/revert
</code></pre>
<p>Revisions are listed newest first. Reverting restores the page content to a specific revision snapshot.</p>
<h2>Public Access</h2>
<p>Consumer sites fetch published pages without authentication:</p>
<pre><code>GET /api/v1/public/sites/:siteSlug/pages/:pageSlug
</code></pre>
<p>Query params:</p>
<ul>
<li><code>lang</code> -- fetch a specific language translation (falls back to site default)</li>
</ul>
<p>Returns the page data with all visible content blocks, ordered by <code>sortOrder</code>.</p>
