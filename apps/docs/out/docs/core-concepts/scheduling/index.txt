9:T96e,<h2>Overview</h2>
<p>Sigil supports time-based content scheduling through <code>publishAt</code> and <code>unpublishAt</code> timestamps on pages. This allows marketing teams to prepare content in advance and have it go live automatically.</p>
<h2>Scheduling API</h2>
<pre><code>PATCH /api/v1/sites/:siteId/pages/:id/schedule
</code></pre>
<h3>Schedule a future publish</h3>
<pre><code class="language-json">{
  "publishAt": "2026-04-01T09:00:00Z"
}
</code></pre>
<p>This sets the page status to <code>scheduled</code>. When the scheduled time arrives, the page transitions to <code>published</code>.</p>
<h3>Schedule publish and unpublish</h3>
<pre><code class="language-json">{
  "publishAt": "2026-04-01T09:00:00Z",
  "unpublishAt": "2026-04-30T23:59:59Z"
}
</code></pre>
<p>The page will be published for exactly one month, then automatically archived.</p>
<h3>Publish immediately</h3>
<pre><code class="language-json">{
  "publishAt": "2026-01-01T00:00:00Z"
}
</code></pre>
<p>A <code>publishAt</code> in the past publishes the page immediately.</p>
<h3>Clear schedule</h3>
<pre><code class="language-json">{
  "publishAt": null,
  "unpublishAt": null
}
</code></pre>
<p>Removes all scheduled transitions.</p>

<h2>Permissions</h2>
<p>Content scheduling requires <code>admin</code> or <code>editor</code> role. Authors can create pages but cannot control their publication schedule.</p>
<h2>Integration with Translations</h2>
<p>Scheduling applies per-translation. You can schedule the English version to publish on April 1 and the Spanish version to publish on April 15.</p>
