9:T1380,<h2>Block Architecture</h2>
<p>Content blocks are the building units of pages. Each block has a <code>blockType</code>, <code>content</code> (JSON), <code>settings</code> (JSON), a <code>sortOrder</code>, and a visibility flag.</p>
<p>Blocks are scoped to pages: <code>/api/v1/sites/:siteId/pages/:pageId/blocks</code>.</p>
<h2>Block CRUD</h2>
<h3>List blocks</h3>
<pre><code>GET /api/v1/sites/:siteId/pages/:pageId/blocks
</code></pre>
<p>Query params: <code>page</code>, <code>limit</code>, <code>blockType</code>, <code>isVisible</code>.</p>
<h3>Create a block</h3>
<pre><code>POST /api/v1/sites/:siteId/pages/:pageId/blocks
</code></pre>
<pre><code class="language-json">{
  "blockType": "hero",
  "content": {
    "headline": "Welcome to Our Site",
    "subheadline": "A brief description.",
    "ctaText": "Get Started",
    "ctaLink": "/signup",
    "alignment": "center"
  },
  "sortOrder": 0,
  "isVisible": true
}
</code></pre>
<h3>Reorder blocks</h3>
<pre><code>PUT /api/v1/sites/:siteId/pages/:pageId/blocks/reorder
</code></pre>
<pre><code class="language-json">{
  "blockIds": ["uuid-1", "uuid-2", "uuid-3"]
}
</code></pre>
