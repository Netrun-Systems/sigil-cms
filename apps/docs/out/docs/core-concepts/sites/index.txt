9:Tb0b,<h2>Multi-Tenant Architecture</h2>
<p>Sigil uses a two-level hierarchy: <strong>tenants</strong> contain <strong>sites</strong>.</p>
<ul>
<li><strong>Tenant</strong> -- an organization or account. Has a plan (<code>free</code>, <code>starter</code>, <code>pro</code>, <code>enterprise</code>), settings, and owns multiple sites.</li>
<li><strong>Site</strong> -- an individual website. Has pages, blocks, media, themes, and a custom domain.</li>
</ul>
<h3>Tenant Schema</h3>
<pre><code class="language-sql">cms_tenants (
  id          UUID PRIMARY KEY,
  name        VARCHAR(100) NOT NULL,
  slug        VARCHAR(50) UNIQUE NOT NULL,
  plan        VARCHAR(20) DEFAULT 'free',  -- free | starter | pro | enterprise
  settings    JSONB DEFAULT '{}',
  is_active   BOOLEAN DEFAULT true,
  created_at  TIMESTAMP,
  updated_at  TIMESTAMP
)
</code></pre>
<h3>Site Schema</h3>
<pre><code class="language-sql">cms_sites (
  id                UUID PRIMARY KEY,
  tenant_id         UUID REFERENCES cms_tenants(id),
  name              VARCHAR(100) NOT NULL,
  slug              VARCHAR(100) NOT NULL,
  domain            VARCHAR(255),          -- custom domain
  default_language  VARCHAR(5) DEFAULT 'en',
  status            VARCHAR(20) DEFAULT 'draft',  -- draft | published | archived
  template          VARCHAR(50),
  settings          JSONB,
  created_at        TIMESTAMP,
  updated_at        TIMESTAMP
)
</code></pre>
<h2>Site CRUD API</h2>
<p>All site endpoints require authentication and tenant context.</p>
<h3>List sites</h3>
<pre><code>GET /api/v1/sites
</code></pre>
<p>Query params: <code>page</code>, <code>limit</code>, <code>status</code>.</p>
<h3>Create a site</h3>
<pre><code>POST /api/v1/sites
</code></pre>
<pre><code class="language-json">{
  "name": "My Brand Site",
  "slug": "my-brand",
  "domain": "mybrand.com",
  "defaultLanguage": "en",
  "status": "draft"
}
</code></pre>
<p>Requires role: <code>admin</code> or <code>editor</code>.</p>
<h3>Custom Domains</h3>
<pre><code>PUT  /api/v1/sites/:id/domain       -- set domain
DELETE /api/v1/sites/:id/domain      -- remove domain
GET  /api/v1/sites/:id/domain/verify -- verify DNS
</code></pre>
<p>Set the body to <code>{ "domain": "mybrand.com" }</code>. The verify endpoint checks DNS configuration.</p>
<h3>Public Site Resolution</h3>
<p>Consumer apps resolve sites by slug or domain:</p>
<pre><code>GET /api/v1/public/sites/:siteSlug/pages     -- list published pages
GET /api/v1/public/sites/by-domain/:domain   -- resolve by custom domain
</code></pre>
<p>No authentication required.</p>
<h2>Tenant Context Middleware</h2>
<p>All authenticated routes pass through <code>tenantContext</code> middleware, which extracts the <code>tenantId</code> from the JWT token and scopes all database queries to that tenant. This prevents cross-tenant data leakage.</p>
