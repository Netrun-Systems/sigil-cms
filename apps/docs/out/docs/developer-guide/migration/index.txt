9:Tb24,<h2>Overview</h2>
<p>The <strong>Migrate</strong> plugin imports existing sites into Sigil from three platforms:</p>
<ul>
<li><strong>WordPress</strong> -- WXR export + REST API</li>
<li><strong>Shopify</strong> -- Admin API + Storefront API</li>
<li><strong>Square Online</strong> -- Catalog API + web scraping</li>
</ul>
<p>No required environment variables -- the plugin is always available.</p>

<h2>Migration Process</h2>
<ol>
<li><strong>Extract</strong> -- pull content from the source platform</li>
<li><strong>Map</strong> -- convert source blocks to Sigil block types</li>
<li><strong>Download media</strong> -- fetch images and rewrite URLs</li>
<li><strong>Preserve SEO</strong> -- carry over meta titles, descriptions, OG images</li>
<li><strong>Import navigation</strong> -- recreate menu structure as page hierarchy</li>
</ol>
<p>Per-item error tracking with retry support ensures partial failures don't block the entire import.</p>
<h2>API</h2>
<p>Migration endpoints are provided by the plugin when loaded. Consult the plugin source at <code>plugins/migrate/src/routes.ts</code> for the full API.</p>
